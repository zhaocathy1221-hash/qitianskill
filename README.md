# Longform Interview Editor Skill

一个用于长篇人物访谈重构的 Codex skill。它适合把长录音、飞书妙记、逐字稿、粗糙访谈笔记和聊天素材，整理成有来源依据、有活人感、有情绪变化和心力的访谈大纲或问答稿。

## 这个 Skill 解决什么问题

它不是把素材总结成观点，也不是把嘉宾修成讲课老师。

它的核心目标是重构一个真实的人：

- 他想要什么；
- 他从什么限制出发；
- 经历了哪些尝试、失败、代价和微小正反馈；
- 判断如何改变；
- 后来的成果为什么成为可能；
- 他最终成为了怎样的人。

## 适合的素材

- 飞书妙记、录音转写、播客逐字稿；
- 人物访谈原始记录；
- 创业者、自由职业者、一人公司、项目复盘类长对谈；
- 聊天记录、粗糙笔记、访谈提纲；
- 已有访谈稿，需要对照原文检查是否写平、写虚、遗漏关键场景。

## 不适合的任务

- 普通帖子发布前润色；
- 精华帖评级、作者反馈、约稿邀约；
- 品牌公关稿、新闻稿、短视频口播稿；
- 没有原始素材依据的虚构人物故事；
- 单纯提炼方法论，而不需要保留人物变化。

如果你的任务是判断一篇内容能不能成为高质量帖子，或要给作者反馈，应该使用单独的内容编辑类 skill，而不是这个访谈重构 skill。

## 安装方式

把 `longform-interview-editor/` 目录复制到你的 Codex skills 目录。

常见位置：

```bash
mkdir -p ~/.codex/skills
cp -R longform-interview-editor ~/.codex/skills/
```

然后重新打开 Codex，或刷新 skill 列表。

## 如何触发

你可以在请求里直接提到 skill 名：

```text
使用 $longform-interview-editor，把这份逐字稿重构成一篇有活人感的人物访谈。
```

也可以描述任务：

```text
这是一份长访谈逐字稿，帮我先建立来源台账，再整理人物变化链条和问答稿。
```

```text
帮我对照原始访谈和成稿，看看哪些地方写平了，哪些关键场景丢了。
```

## 推荐工作流

1. 取得完整逐字稿，保留说话人和时间戳。
2. 建立来源台账，标记事实、数字、原话、场景、情绪、决策和敏感信息。
3. 找人物变化链条：想要的生活 -> 起点限制 -> 尝试 -> 失败或代价 -> 微小正反馈 -> 能力或信心 -> 再次尝试 -> 可见成果。
4. 选择唯一主线：人物线、项目线或团队线。
5. 产出主题大纲。
6. 重构问答稿。
7. 做最终审查：来源覆盖、数字口径、人物归属、敏感信息、方法内容是否淹没人物。

如果你手里是录音、飞书妙记或飞书文档，按 [从录音到访谈稿：使用说明书](HOW_TO_USE_FROM_RECORDING.md) 一步步处理。

## 不用 Skill 时怎么用

如果你不用 Codex skill，也可以直接使用 [prompts.md](prompts.md) 里的提示词。

建议顺序：

1. 总控提示词；
2. 来源台账提示词；
3. 人物变化链条提示词；
4. 主题大纲提示词；
5. 问答稿重构提示词；
6. 最终审查提示词。
