# 长篇人物访谈重构

## 目标

重构一个真实的人，而不是总结一个主题。写清楚他想要什么、经历了什么失败、哪些微小结果支撑他继续向前、判断如何改变，以及最终成果付出了什么代价。

把“给读者补足心力”作为重要目标。但心力必须来自有来源依据的场景和可信的变化过程，不能依靠励志口号或虚构情绪。

## 明确能力边界

本 Skill 只用于从原始素材到访谈稿的重构：

- 清理逐字稿，统一说话人；
- 建立来源台账与证据卡片；
- 梳理人物主线、项目支线与主题大纲；
- 重构问答，撰写长篇人物访谈；
- 检查来源覆盖、素材遗漏、数字口径和人物归属；
- 根据用户反馈局部修改访谈稿。

不要用它判断帖子能否评为精华、撰写作者反馈、分配内容团队资源或进行平台发布前的最终润色。访谈初稿完成后，再交给对应的内容改稿 Skill 处理这些任务。

## 必须读取的参考文件

只读取当前任务需要的内容：

- 开始拟大纲或写稿前，读取 [访谈编辑标准](references/interview-editorial-standard.md)。
- 处理长逐字稿或对照原文与成稿前，读取 [来源台账规范](references/source-ledger-schema.md)。
- 修改核心规则或准备发布新版 Skill 前，读取 [评测用例](references/evaluation-suite.md)。

## 工作流程

### 1. 整理原始素材

用户提供录音或妙记链接时，优先取得完整逐字稿。保留说话人标签和时间戳。只有在上下文足够明确时才修正语音识别错误；无法确定的内容标记为待确认。

将用户本次需要的输出和原始素材分开记录：

- 只要主题大纲；
- 完整问答稿；
- 单个章节；
- 原文与成稿对照审查；
- 修改已有访谈稿。

不要因为提示词描述了最终文章形式，就误以为用户当前一定要求产出全文。

### 2. 建立来源台账

按照说话轮次和完整事件处理长素材，不要按固定字数粗暴切割。为每段素材设置稳定编号，并记录时间戳或原文位置。

分别提取以下卡片：

- 事实与数字；
- 原话与口述特征；
- 当下生活场景；
- 欲望、目标与现实压力；
- 决策过程与判断形成；
- 失败、代价、怀疑与矛盾；
- 微小的正反馈；
- 后来可见的成果；
- 可以独立形成项目线的方法内容；
- 需要编辑确认的敏感人生经历。

具体格式遵循 [来源台账规范](references/source-ledger-schema.md)。

### 3. 找到人物变化链条

当原始素材能够支撑时，重构以下链条：

`想要的生活 → 起点限制 → 第一次尝试 → 失败或代价 → 微小正反馈 → 获得信心或能力 → 再次尝试 → 可见成果 → 成为现在的自己`

不要因为某个项目最终失败或上限不高就删掉它。如果它解释了现金流、信心、产品感、平台经验，或者解释了人物为什么愿意再次尝试，就应该保留。

### 4. 确定唯一主线

拟大纲前先选择：

- **人物线**：选择、挫折、情绪变化、身份变化和想要的生活；
- **项目线**：机会、执行、案例、运营过程和可复制方法；
- **团队线**：授权、招聘、标准流程、管理和创始人逐渐退出具体事务。

如果人物经历和项目方法都足够丰富，建议拆成两篇文章，不要让教程内容淹没人物。

### 5. 产出主题大纲

每个章节都应包含：

- 本章作用；
- 本章推进的一个变化或问题；
- 对应的来源编号或时间戳；
- 关键场景或原话；
- 由事件支撑的情绪；
- 敏感信息或缺失信息；
- 有意移入支线文章的素材。

章节标题用于推进故事，不用于制造短视频式噱头。

### 6. 保持口述连续性地重构问答

重构问题只能用于澄清嘉宾确实回答过的内容。不能通过写出一个更聪明的问题，虚构出原文里不存在的回答。

当用户希望问答节奏更轻快时，单轮回答默认控制在 150—300 个汉字。完整场景、决策过程或情绪转折如果会因此受损，可以放宽到 300—500 字。应在事件、原因、决策或时间节点处拆分，不能从一个完整念头的中间切断。

保留有意义的口述特征：

- 自我修正；
- 不确定感；
- 笑声或尴尬；
- 重复强调；
- 粗粝但有辨识度的表达。

删除无意义口头禅和语音识别噪音，但不要把人物修成一个讲课老师。

### 7. 采用局部补丁式修改

将来源台账和已经确认的大纲视为稳定的中间成果。

当用户要求加强某一章的情绪、细节或人物感时：

- 只修改受影响的章节；
- 保持无关章节稳定；
- 说明补回或删除了哪些来源卡片；
- 有比较价值时保留上一版本。

只有用户明确要求更换整体结构时，才重新生成全文。

### 8. 完成最终审查

检查：

- 每项重要事实、原话、数字和情绪是否能对应原始来源；
- 高度专注的工作时间与全天总工作时间是否区分；
- 峰值、稳定值、累计值、GMV、营收和利润是否混淆；
- 说话人姓名与栏目展示名称的对应关系是否明确；
- 敏感人生经历是否已确认、说明理由后删除或标记待确认；
- 章节编号和分隔符是否统一；
- 稿件是否包含欲望、失败、微小成果和可见变化；
- 方法内容是否淹没了人物；
- 被删掉的高价值素材是否进入遗漏清单。

稿件为 Markdown 时，运行 `scripts/validate_interview_draft.py <稿件.md>` 完成机械检查。

## 输出约定

### 主题大纲

依次输出：

1. 一句话主题；
2. 人物变化链条；
3. 带来源标记的章节大纲；
4. 被移到支线的项目或团队素材；
5. 敏感信息与待补信息；
6. 标题方向。

### 完整访谈稿

先给出访谈正文，再附一份简短审查记录：

- 原始素材覆盖情况；
- 有意删除的内容；
- 需要确认的事实或措辞；
- 数字口径；
- 哪些问题由编辑重构。

### 原文与成稿对照审查

依次输出：

1. 成稿保留得好的内容；
2. 被写平或过度方法化的部分；
3. 缺失的场景、微小成果与情绪因果；
4. 缺乏原文依据的新增内容或人物归属问题；
5. 按优先级排列的修改方案。
