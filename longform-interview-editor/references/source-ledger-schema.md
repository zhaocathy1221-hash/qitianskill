# 来源台账规范

使用 `S01`、`S02` 等稳定编号，并保留原始时间戳或区块位置。

## 素材卡片

```text
编号：S01
来源：00:43—02:10 或区块编号
说话人：嘉宾姓名
事件或主题：现在的晨间生活
事实：7:00 起床；7:30 跑步；正在准备半程马拉松
场景：跑完喝咖啡；戴着耳机听音乐走回家
直接原话：只保留简短且有来源的表达
情绪：嘉宾明确表达开心和平静
决策或判断：把阅读放在工作前，因为拖到后面往往无法完成
人物弧线作用：现在的生活状态／开头的反差
敏感程度：低
使用状态：主线｜支线｜删除｜待确认
```

## 数字台账

每个可能发布的数字都要记录：

```text
表述：每天工作两小时
定义：高度专注的生产时间
相关表述：包含运营、学习和杂事时，每天共四至五小时
时间范围：现在的日常状态
峰值／稳定／累计：当前典型估算
来源：时间戳或区块编号
标题能否使用：等待口径一致性检查
```

分别记录 GMV、营收、利润、实际到账、工资、订单量、团队人数、账号数量、工作时间和统计周期。


## 覆盖与遗漏清单

为每张高价值卡片标记一种状态：

- `已使用`：出现在成稿中；
- `已合并`：与其他素材合并，但没有损失因果关系；
- `支线文章`：有价值，但属于另一条主线；
- `重复删除`：没有独特信息；
- `敏感删除`：等待确认或确认后不使用；
- `待澄清`：原始素材不清楚或互相矛盾。

遗漏清单可以避免编辑长逐字稿时，悄悄删掉“这个人为什么还能继续”的关键原因。

## 说话人对应表

分别记录原始素材里的真实说话人和文章展示名称：

```text
原始说话人：实际采访者姓名
展示名称：媒体、栏目或账号名称
对应依据：用户明确提供｜尚未确认
```

没有明确依据时，不得替换采访者署名。
