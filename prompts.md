# 长篇人物访谈提示词库

基于 `longform-interview-editor` 整理。适用于长录音、飞书妙记、逐字稿、粗糙访谈笔记、聊天素材到人物访谈大纲或问答稿的重构。

## 1. 总控提示词

```text
你是一个长篇人物访谈编辑。你的任务不是总结主题，而是重构一个真实的人。

请基于我提供的原始素材，写清楚：
1. 这个人想要什么；
2. 他从什么限制和处境出发；
3. 经历了哪些尝试、失败、代价和微小正反馈；
4. 他的判断如何形成和改变；
5. 后来的成果为什么成为可能；
6. 他最终成为了怎样的人。

重要原则：
- 所有事实、数字、原话和情绪必须有原始素材依据。
- 不要虚构人物内心活动，不要把编辑推断写成嘉宾第一人称。
- 不要把人物修成讲课老师，要保留有意义的口述感、自我修正、不确定感和粗粝表达。
- 不要用励志总结弥补人物路径缺失。心力必须来自真实场景、失败、微小成果和变化过程。
- 如果项目方法内容过多，请判断它是否应该拆成支线文章，不要让教程淹没人物。

请先判断本次素材更适合以下哪条主线：
- 人物线：选择、挫折、情绪变化、身份变化和想要的生活；
- 项目线：机会、执行、案例、运营过程和可复制方法；
- 团队线：授权、招聘、标准流程、管理和创始人逐渐退出具体事务。

输出时，请明确区分：
- 直接原话；
- 忠实转述；
- 编辑推断；
- 待确认信息。
```

## 2. 逐字稿清理提示词

```text
请清理下面这份访谈逐字稿。

要求：
1. 保留说话人标签和时间戳。
2. 删除无意义口头禅、重复语气词、明显语音识别噪音。
3. 只有在上下文足够明确时，才修正语音识别错误。
4. 无法确定的内容用「[待确认：...]」标记，不要擅自补全。
5. 保留有意义的口述特征，包括自我修正、不确定感、笑声、尴尬、重复强调、有辨识度的表达。
6. 不要改写成正式书面稿，只做可读性清理。

输出格式：
- 按时间顺序输出清理后的逐字稿；
- 最后列出「待确认内容」。
```

## 3. 来源台账提示词

```text
请基于下面的原始访谈素材，建立来源台账。

请按照说话轮次和完整事件处理，不要按固定字数粗暴切割。为每段高价值素材设置稳定编号，如 S01、S02、S03。

每张素材卡片包含：
- 编号；
- 来源：时间戳或原文区块位置；
- 说话人；
- 事件或主题；
- 事实；
- 场景；
- 直接原话；
- 情绪；
- 决策或判断；
- 人物弧线作用；
- 敏感程度；
- 使用建议：主线 / 支线 / 删除 / 待确认。

请重点提取：
1. 事实与数字；
2. 原话与口述特征；
3. 当下生活场景；
4. 欲望、目标与现实压力；
5. 决策过程与判断形成；
6. 失败、代价、怀疑与矛盾；
7. 微小正反馈；
8. 后来可见的成果；
9. 可独立形成项目线的方法内容；
10. 需要确认的敏感人生经历。

最后请补充：
- 数字台账；
- 人物变化台账；
- 高价值素材遗漏风险清单。
```

## 4. 人物变化链条提示词

```text
请从原始素材中提炼人物变化链条。

请不要只总结结果，而要追踪这个人如何一步步变成现在的样子。

请按这个结构输出：
1. 想要的生活；
2. 起点限制；
3. 第一次尝试；
4. 失败或代价；
5. 微小正反馈；
6. 获得的能力或信心；
7. 再次尝试；
8. 后来的成果；
9. 成为现在的自己。

要求：
- 每一环都标注对应来源编号或时间戳。
- 如果某一环素材不足，请标记「缺失」或「待确认」。
- 不要因为某个项目最终失败或上限不高就删掉它；如果它提供了现金流、信心、产品感、平台经验或再次尝试的能量，就应该保留。
```

## 5. 主题大纲提示词

```text
请基于来源台账和人物变化链条，产出一份长篇人物访谈主题大纲。

请先选择唯一主线：
- 人物线；
- 项目线；
- 团队线。

如果人物经历和项目方法都足够丰富，请建议拆成两篇文章，并说明：
- 人物篇写什么；
- 项目篇写什么；
- 哪些素材移入支线。

每个章节请包含：
1. 章节标题；
2. 本章作用；
3. 本章推进的一个变化或问题；
4. 对应来源编号或时间戳；
5. 关键场景或原话；
6. 由事件支撑的情绪；
7. 敏感信息或缺失信息；
8. 有意移入支线文章的素材。

整体输出格式：
1. 一句话主题；
2. 人物变化链条；
3. 带来源标记的章节大纲；
4. 被移到支线的项目或团队素材；
5. 敏感信息与待补信息；
6. 标题方向。

章节标题要推进故事，不要制造短视频式噱头。
```

## 6. 问答稿重构提示词

```text
请把下面的访谈素材重构为一篇长篇人物访谈问答稿。

核心目标：
写出一个真实的人，而不是整理一套方法论。请让读者看到他的欲望、限制、失败、微小正反馈、判断变化和最终结果。

问答规则：
1. 问题可以由编辑重构，但只能用于澄清嘉宾确实回答过的内容。
2. 不要通过写出更聪明的问题，虚构出原文里不存在的回答。
3. 回答默认控制在 150-300 字。
4. 如果完整场景、决策过程或情绪转折会受损，可以放宽到 300-500 字。
5. 拆分回答时，应在事件、原因、决策或时间节点处拆分，不要从一个完整念头中间切断。

语言规则：
- 删除无意义口头禅和识别噪音；
- 保留有意义的口述感、自我修正、不确定感、笑声或尴尬；
- 不要把嘉宾修成讲课老师；
- 不能把编辑推断写成嘉宾第一人称；
- 不要虚构焦虑、兴奋、崩溃等情绪。

事实规则：
- 所有事实、数字、原话、情绪必须能对应原始来源；
- 区分 GMV、营收、利润、实际到账、订单量、团队人数、账号数量、工作时间和统计周期；
- 区分峰值、稳定值和累计值；
- 区分高度专注工作时间和全天总工作时间；
- 敏感人生经历必须标记为待确认。

输出：
1. 访谈正文；
2. 简短审查记录，包括：
   - 原始素材覆盖情况；
   - 有意删除的内容；
   - 需要确认的事实或措辞；
   - 数字口径；
   - 哪些问题由编辑重构。
```

## 7. 原文与成稿对照审查提示词

```text
请对照原始访谈素材和已有成稿，做一次访谈稿审查。

审查重点：
1. 成稿保留得好的内容；
2. 被写平、写虚或过度方法化的部分；
3. 缺失的具体场景、微小成果与情绪因果；
4. 缺乏原文依据的新增内容；
5. 人物归属或说话人归属是否有问题；
6. 数字口径是否混淆；
7. 是否删掉了支撑人物继续往前走的关键素材；
8. 方法内容是否淹没人物主线。

请按优先级输出修改建议：
- P0：事实错误、无来源内容、数字口径问题；
- P1：人物主线断裂、关键因果缺失；
- P2：语言过度书面化、口述感损失、节奏问题；
- P3：标题、章节顺序、表达优化。

请不要直接重写全文，除非我明确要求。先给出诊断和修改方案。
```

## 8. 局部修稿提示词

```text
请只修改下面访谈稿中的指定章节，不要重写全文。

修改目标：
【在这里填写：加强情绪 / 补充场景 / 缩短回答 / 增强人物感 / 降低方法论感 / 调整数字口径 / 删除敏感内容】

要求：
1. 只修改受影响章节，保持无关章节稳定。
2. 只能补入有来源依据的场景、原话、事实和情绪。
3. 不要虚构人物内心。
4. 如果素材不足，请标记「待补充」。
5. 修改后说明：
   - 改动了哪些段落；
   - 补回或删除了哪些来源编号；
   - 哪些内容仍需确认。
```

## 9. 最终审查提示词

```text
请对这篇访谈稿做最终发布前审查。

请逐项检查：
1. 每项重要事实、原话、数字和情绪是否能对应原始来源；
2. 高度专注工作时间与全天总工作时间是否区分；
3. 峰值、稳定值、累计值是否混淆；
4. GMV、营收、利润、到账、订单量是否混淆；
5. 说话人姓名与栏目展示名称是否明确；
6. 敏感人生经历是否已确认、删除或标记待确认；
7. 章节编号和分隔符是否统一；
8. 稿件是否包含欲望、失败、微小成果和可见变化；
9. 方法内容是否淹没人物；
10. 被删掉的高价值素材是否进入遗漏清单。

请输出：
- 通过项；
- 风险项；
- 必须修改项；
- 可选优化项；
- 待用户确认项。
```

## 10. 轻量版一键提示词

```text
请把我提供的访谈逐字稿，整理成一篇有活人感的长篇人物访谈。

请不要只总结方法，而要写出这个人：
- 想要什么；
- 从哪里开始；
- 经历过哪些失败、代价和微小正反馈；
- 判断如何变化；
- 为什么后来能做成；
- 现在变成了怎样的人。

要求：
- 所有事实、数字、原话、情绪都要来自原始素材；
- 不虚构心理活动；
- 保留关键场景和有辨识度的口述表达；
- 项目方法太多时，主动拆成支线；
- 敏感信息标记待确认；
- 输出包括：一句话主题、人物变化链条、章节大纲、访谈正文、审查记录。
```
